### Запуск нового потока
Есть 2 способа: 
1) Отнаследоваться от Thread и переопределить run(), потом в главном потоке создать экземпляр созданного класса и на экземпляре вызвать start() 
2) Имплементировать интерфейс Runnable и реализовать run(), потом в главном потоке передать в конструктор класса Thread объект реализующий Runnable
!!!Второй способ предпочтительнее из-за ограничения множественного наследования в java!!!

Запуск потоков всегда делать через метод thread.start()

### Остановка потока
Никогда не вызывать thread.stop() т.к. мгновенно останавливается поток и данные могут повредиться
Вместо этого вызывать thread.interrupt(), он только устанавливает значение isInterrupted в true

Thread.currentThread() - ссылка на текущий поток. Можно получить его состояние(в т.ч. isInterrupted)

### Сон потока на заданное время
Thread.sleep(ms)

### Дождаться выполнения дочернего потока в главном
Метод thread.join() позволяет в вызывающем потоке дождаться завершения дочернего потока 
join() - останавливает главный поток, поэтому обязательно обернуть в try-catch

### Порядок запуска потоков не гарантирован
Никогда не ориентироваться на порядок запуска!!!
