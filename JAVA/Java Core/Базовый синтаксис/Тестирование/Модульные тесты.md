Для написания модульных тестов как правило используется специальный [фреймворк](JAVA/Java%20Core/Базовый%20синтаксис/_Термины.md#Фреймворк). Например [JUnit 5](JAVA/Java%20Core/Базовый%20синтаксис/_Библиотеки.md#org.junit.jupiter.api.Test).

У нас в проекте используется система [Maven](JAVA/Java%20Core/Базовый%20синтаксис/_Термины.md#Maven), вся конфигурация проекта находится в файле pom.xml, он находится в корне проекта. Благодаря наличию этого конфигурационного файла, произошло автоматическое подключение фреймворка JUnit5 к проекту. Термин "подключен к проекту" означает, что код из этого фреймворка можно будет использовать в нашем проекте.

Фреймворк JUnit5 позволяет создавать независимые блоки кода для тестирования.
При использовании JUnit5 мы можем писать независимые [методы](JAVA/Java%20Core/Базовый%20синтаксис/_Термины.md#Метод), которые можно запускать и проверять отдельно.

Есть общее [соглашение](JAVA/Java%20Core/Базовый%20синтаксис/4.%20Правила%20оформления%20кода.md#Модульные%20тесты) о создании JUnit тестов. 

IntelliJ Idea позволяет сгенерировать [класс-тест](INTELLIJ%20IDEA/Хоткеи.md#Alt%20+%20Insert%20-%20генерация%20класс-теста). Для этого:
1. Встаём курсором на имени класса
2. Жмём Alt+ Insert
3. Выбираем Test
4. Выбираем библиотеку JUnit5
5. Отмечаем методы которые хотим протестировать

Над каждым методом ставится [аннотация](JAVA/Java%20Core/Базовый%20синтаксис/_Термины.md#Аннотация) @Test
