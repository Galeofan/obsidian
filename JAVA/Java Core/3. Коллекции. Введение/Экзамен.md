
1. **Что такое коллекция?**
Коллекция в Java — это объект, который служит контейнером для хранения группы элементов, таких как объекты или примитивные типы (через обёртки). Она позволяет удобно добавлять, удалять, упорядочивать и обрабатывать данные.
Коллекции находятся в пакете **`java.util`**, и Java предоставляет мощный набор интерфейсов и классов для работы с ними.

    1.1 Основные методы интерфеса коллекций:
		boolean add(E e)
		boolean addAll(Collection<? extends E> c)
		Iterator<E> iterator() – Возвращает объект Iterator, который позволяет обойти элементы коллекции при помощи цикла for each;
		boolean removeAll(Collection<?> c)
		boolean retainAll(Collection<?> c) - метод, который позволяет сохранить в коллекции только те элементы, которые содержатся в коллекции, которая была передана в метод

		int size()
		boolean isEmpty()
		boolean contains(Object o) – метод возвращает true, если переданный в метод объект содержится в коллекции.
		void clear()
		Object[] toArray() - метод возвращает массив, который содержит все элементы, содержащиеся в коллекции у которой был вызван этот метод.

2. **Назовите преимущества использования коллекций по сравнению с массивами.**
- Хранение данных: Можно хранить большое количество элементов без необходимости заранее знать их количество.
- Возможность хранить значения без дублей и без null;
- Возможность хранить разнородные данные (поддержка Дженериков);
- Гибкость: Коллекции легко адаптируются для разных задач: работа с уникальными элементами, поддержание порядка, сортировка и т. д.
- Упрощение работы: Java предоставляет готовые методы для добавления, удаления, поиска и обработки данных.

3. **Какие данные могут хранить коллекции?**
- простые последовательные наборы элементов;
- наборы пар «ключ-значение».
При этом только ссылочные типы и обёртки.

4. **Какие есть типы коллекций? Как они характеризуются? Расскажите про иерархию коллекций List, Set, Map.**
- Списки (List) - позволяют хранить одно и то же значение несколько раз;
- Наборы (Set) - неупорядоченный набор уникальных элементов;
- Очереди (Queue) - позволяют получать элементы в том же порядке, в котором они были помещены в очередь. Принцип FIFO;
- Словари (Map) - хранят пару ключ-значение;
### Основные интерфейсы коллекций:

1. **`Collection`**  
    Основной интерфейс, который описывает базовые методы для работы с коллекциями (например, добавление, удаление, проверка размера).  
    Подразделяется на три главных типа:
    
    - **`List`**
    - **`Set`**
    - **`Queue`**
2. **`Map`**  
    Отдельный интерфейс для работы с парами "ключ-значение". В отличие от `Collection`, здесь используются ключи для доступа к данным.
    

---

### Виды коллекций:

1. **`List`** (список)  
    Упорядоченная коллекция, где элементы хранятся в определённой последовательности. Допускаются дубликаты.
    
    - **Примеры классов:** `ArrayList`, `LinkedList`, `Vector`.
    - **Пример:**
        
        java
        
        Копировать код
        
        `List<String> list = new ArrayList<>(); list.add("яблоко"); list.add("банан"); list.add("яблоко"); System.out.println(list); // [яблоко, банан, яблоко]`
        
2. **`Set`** (множество)  
    Коллекция, которая хранит только уникальные элементы. Порядок не гарантируется (в зависимости от реализации).
    
    - **Примеры классов:** `HashSet`, `LinkedHashSet`, `TreeSet`.
    - **Пример:**
        
        java
        
        Копировать код
        
        `Set<String> set = new HashSet<>(); set.add("яблоко"); set.add("банан"); set.add("яблоко"); System.out.println(set); // [яблоко, банан] (без дубликатов)`
        
3. **`Queue`** (очередь)  
    Коллекция для работы с элементами по принципу "первый пришёл — первый вышел" (FIFO).
    
    - **Примеры классов:** `PriorityQueue`, `LinkedList`.
    - **Пример:**
        
        java
        
        Копировать код
        
        `Queue<String> queue = new LinkedList<>(); queue.add("задача1"); queue.add("задача2"); System.out.println(queue.poll()); // задача1`
        
4. **`Map`** (словарь)  
    Коллекция для хранения данных в виде пар "ключ-значение".
    
    - **Примеры классов:** `HashMap`, `TreeMap`, `LinkedHashMap`.
    - **Пример:**
        
        java
        
        Копировать код
        
        `Map<String, Integer> map = new HashMap<>(); map.put("яблоко", 3); map.put("банан", 5); System.out.println(map); // {яблоко=3, банан=5}`

### Преимущества использования коллекций:

- **Гибкость:** Вы можете выбрать нужную коллекцию под задачу.
- **Экономия времени:** Не нужно разрабатывать собственные алгоритмы для работы с данными.
- **Удобство:** Простота в использовании благодаря множеству готовых методов (`add()`, `remove()`, `sort()`, и т.д.).

---

### Различие между коллекцией и массивом:

|**Критерий**|**Коллекция**|**Массив**|
|---|---|---|
|**Размер**|Динамический, меняется во время работы программы.|Фиксированный, задаётся при создании.|
|**Тип данных**|Может хранить только объекты (но есть автоупаковка для примитивов).|Может хранить примитивы и объекты.|
|**Гибкость**|Имеет множество встроенных методов для удобной работы.|Требует ручного написания многих операций.|

### Когда использовать коллекции?

- Когда нужно работать с динамическим количеством данных.
- Если требуется уникальность элементов (`Set`).
- Для поиска по ключу (`Map`).
- Для упорядочивания элементов (`List` или `TreeSet`).
