4. **Prototype** - Проверить концепцию UX/UI

7. **Production Release (GA – General Availability)** - Полный промышленный запуск
8. **Post-GA / Scaling / Iteration** - Развитие и рост
9. End-of-Life (Sunset) - Завершение
# 1️⃣ Idea / Problem Statement

Формулируется проблема:
- что делаем,
- зачем,
- для кого,
- какую боль решаем.

**Вывод:** документ с формулировкой ценности.

---

# 2️⃣ Discovery / Research (исследование)

Задача — понять:
- как люди делают это сейчас,
- какие уже есть решения,
- ограничения, риски, зависимости,
- архитектурные варианты.

**Вывод:** концепт + список требований высокого уровня.

---

# 3️⃣ PoC — Proof of Concept (“доказательство работоспособности”)

Цель — проверить **можно ли это сделать технически**.
Характерные признаки:
- нет UI,
- нет логики,
- нет безопасности,
- всё может быть "костылями",
- код выбрасывается.

**Типичный пример:** протестировали API стороннего провайдера, убедились что OTP приходит.

**Вывод:** техническая возможность доказана.

---

# 4️⃣ Prototype (“интерактивная болванка”)

Первый рабочий набросок интерфейса или сценария.
Разница с PoC:
- PoC = доказательство, что _работает_
- Prototype = доказательство, что _удобно и понятно пользователю_

**Вывод:** показано, как продукт будет выглядеть.

---

# 5️⃣ MVP — Minimum Viable Product

Минимально жизнеспособная версия для реальных пользователей.
Характерные особенности:
- только критические фичи,
- базовая безопасность,
- базовый мониторинг,
- минимальная архитектура но без долгов.

**Важно:** MVP — **не PoC**, а “версия 1.0-lite”.

**Вывод:** продукт решает ключевую задачу и приносит ценность.

---

# 6️⃣ Pilot / Beta

Используется ограниченной группой:
- тестовые пользователи,
- фича-флаг,
- отдельный клиент.

Цель — поймать реальные проблемы до большого продакшена.

**Вывод:** продукт работает в реальных условиях.

---

# 7️⃣ Production Release (GA — General Availability)

Запуск на всех:
- SLA,
- логирование,
- мониторинг,
- отказоустойчивость,
- безопасность,
- резервирование.

Здесь требования самые строгие.

**Вывод:** это полноценный промышленный сервис.

---

# 8️⃣ Post-GA / Iteration / Scaling

Этап развития и масштабирования:
- улучшения,
- новые фичи,
- оптимизация,
- A/B тесты,
- улучшение UX,
- увеличение нагрузки.

---

# 9️⃣ End-of-Life (Sunset)

Закрытие продукта:
- миграция пользователей,
- архивирование данных,
- отключение инфраструктуры.
